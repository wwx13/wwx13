# AMOS

# 整体结构

# 两个目标函数

  1. 生成器损失只关心MASK的预测正误
  2. 判别器是关心全序列的替换检测正误，虽然只在mask位置可能有替换其余都是未替换的。

# 模拟多encoder的多层mlm分类

实现是按照对应的layer截断输入梯度进行的。

# 引入不属于编码器和解码器的权重参数v

一个mlm分类层包含一个映射参数两个操作：
0. 输入是encoder顶层多头输出的特征:: f
1.对transformer encoder的输出进行映射，映射到embedding weight的hidden维度:: h
2.随后和embedding weight进行乘法操作，映射到词表分类空间，最后softmax操作。

TIP

这一步实际对梯度进行截断。防止判别器损失影响任务mlm自己私有参数h。

因此：

实际mlm会更新生成器和自己私有参数h

rpd任务会影响判别器参数，生成器参数（这个通过f和embedding weight更新做为入口影响其余生成器参数），额外的层分布权重v。

特别的：rpd通过未mask的token对应的embedding weight 和 mask位置的embedding weight 和 gabmbel以后的加权分布影响generator和v。

总结：1个地方梯度截断，1个的地方gambel使得mask的地方的特征能被判别器反向传播更新。

# 参数共用

generator和discriminator共享embedding相关参数 各自分开有自己的transformer layer参数

# gumble softmax

按logit概率进行取argmax转onehot,是会截断梯度的因此通过

def gumbel_softmax(logits, temperature=1, hard=False):
    拟onehot = gumbel_softmax获取拟onehot(logits, temperature)
    if not hard:  拟onehot
    真onehot = 拟onehot转真onehot(拟onehot)
    真onehot = (真onehot - 拟onehot).detach() + 拟onehot
    return 真onehot

对于小的temperature，样本接近于one-hot，但梯度的方差很大 对于大的temperature，样本是平滑的，但梯度的方差很小（不同的category区分力度小）
