# 调整计算的顺序和计算函数

适用场景： 当序列长度很长时，模型计算attention速度很慢 设n>>d

  1. QKV的计算顺序： 先计算QK： nn 代表每个时刻对各个时刻的相关程度，再计算V 得到nd. Q的复杂度为n2， n个向量每个需要和n个向量乘。 先计算KV, 得到dd 复杂度为 nd 接着Q, 复杂度d2 ,整体O（n）。
  2. 但是softmax(QK/\sqrt{d}) s这个使得不能简单先计算后面，因为标准的 softmax是基于exp计算的，我们需要一直计算方式使得， 可以拆解QK的相似度。 有三种方式。 Transformers are RNNs: Fast Autoregressive Transformers with Linear Attention

# 稀疏attention

OpenAI的Sparse Attention，通过“只保留小区域内的数值、强制让大部分注意力为零”的方式，来减少Attention的计算量。经过特殊设计之后，Attention矩阵的大部分元素都是0，因此理论上它也能节省显存占用量和计算量。后续类似工作还有《Explicit Sparse Transformer: Concentrated Attention Through Explicit Selection》、《Longformer: The Long-Document Transformer》等

# 多头的表达能力改进

适用场景：数据量充分； 文本很长

  1. 增大key_size 由于原始的multi-head attention 的Q,K参数量实际少于预期 表征的参数量，因此存在Q,K表征瓶颈。 第一种方式是在得到Q,K的线性层处增大维度，而V的维度不变，这使得multi head拼接的最终维度 依然和改进以前完全一样。 Low-Rank Bottleneck in Multi-head Attention Models (opens new window)
    最后，附上我们预训练的两个增大了key_size的RoBERTa小模型，欢迎大家使用（我们称之为RoBERTa+）：

https://github.com/ZhuiyiTechnology/pretrained-models 2. 不增大key size, 我们考虑通过引入额外的参数把多个头的 QK结果进行混合，这么做的原因有两个：1. 每个头反应在一个位置 QK结果反应的是不同角度的相关性，因此彼此不应该独立，应该有关联。 2. 新的 混合参数引入了更多可能，使得Q*K的表达能力增强。 alking-Heads Attention (opens new window)
